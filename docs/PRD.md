## 一、产品定位

### 1.1 核心问题

| 痛点 | 现状 |
|------|------|
| C盘空间不足 | 用户不知道什么占空间 |
| 清理风险 | 怕删错东西，不敢动手 |
| 迁移复杂 | 符号链接、Junction 概念晦涩 |
| 反复出现 | 清理后不久又满了 |

### 1.2 目标用户

| 用户类型 | 特征 | 核心诉求 |
|---------|------|---------|
| 普通用户 | 不懂技术，C盘红了就慌 | 一键清理，安全可靠 |
| 开发者 | 装各种环境，目录复杂 | 精准迁移，保留数据 |
| 运维人员 | 管理多台机器 | 批量操作，可脚本化 |

### 1.3 产品形态

```
┌─────────────────────────────────────────────┐
│              Disk Space Manager             │
├─────────────────────────────────────────────┤
│  CLI        │ 命令行工具，开发者/运维使用     │
│  Desktop App│ 桌面应用，普通用户使用          │
│  MCP Server │ AI 集成，让 AI 帮用户操作       │
│  API        | 第三方集成                      │
└─────────────────────────────────────────────┘
```

## 三、AI 模块设计

### 3.1 为什么需要 AI？

| 场景 | 规则引擎 | AI |
|------|---------|-----|
| "npm-cache 可以删" | ✅ 能判断 | ✅ 能判断 |
| "AppData\Local\某奇怪的目录" | ❌ 规则覆盖不到 | ✅ 根据目录名+文件类型推断 |
| "用户说我是做 Python 开发的" | ❌ 无法理解上下文 | ✅ 保留 Python 相关目录 |
| "这个目录看起来像项目代码" | ❌ 难以判断 | ✅ 分析文件结构推断 |
| 向用户解释为什么 | ❌ 只能输出固定文案 | ✅ 自然语言解释 |

### 3.2 AI 调用点

```
扫描完成后 → 调用 AI → 分析每个目录 → 返回结构化建议
```

**输入给 AI：**

```json
{
  "directories": [
    {"path": "AppData/Local/npm-cache", "size_mb": 1200, "file_types": {".json": 50, ".txt": 30}},
    {"path": "AppData/Local/xxx_unknown", "size_mb": 800, "file_types": {".pyc": 100, ".py": 20}}
  ],
  "user_context": "我是 Python 开发者，主要用 VSCode",
  "target_drive": "D:\\"
}
```

**AI 返回：**

```json
{
  "recommendations": [
    {
      "path": "AppData/Local/npm-cache",
      "action": "clean",
      "confidence": 0.95,
      "risk": "low",
      "reason": "npm 缓存目录，可安全清理，不影响已安装的包"
    },
    {
      "path": "AppData/Local/xxx_unknown",
      "action": "migrate",
      "confidence": 0.7,
      "risk": "medium",
      "reason": "该目录包含大量 .pyc 文件，可能是 Python 缓存，建议迁移而非删除",
      "target": "D:\\migrated\\xxx_unknown"
    }
  ],
  "summary": "可释放约 2GB 空间，建议优先清理缓存类目录"
}
```

### 3.3 模型选择

| 模型 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| OpenAI GPT-4o | 效果好，API 稳定 | 贵，国内访问需代理 | 高端付费版 |
| DeepSeek | 便宜，国内可用 | 效果略差 | 国内免费版 |
| 本地模型 (Ollama) | 无需联网，隐私 | 效果一般，需硬件 | 企业私有部署 |
| 混合模式 | 灵活 | 复杂 | 商业版 |

### 3.4 AI 配置项

```yaml
ai:
  provider: "deepseek"  # openai / deepseek / ollama / custom
  model: "deepseek-chat"
  api_key: "${DISK_MANAGER_API_KEY}"  # 环境变量或配置文件
  base_url: "https://api.deepseek.com"  # 可自定义
  temperature: 0.3  # 低温度保证稳定输出
  timeout: 30
  
  # 可选：本地模型
  ollama:
    model: "qwen2.5:14b"
    base_url: "http://localhost:11434"
```

## 四、商业设计

### 4.1 定价策略

| 版本 | 价格 | 功能 |
|------|------|------|
| **免费版** | ¥0 | 基础扫描 + 规则推荐（无AI） |
| **个人版** | ¥9.9/月 | AI 分析 + 一键清理 + 迁移 |
| **专业版** | ¥29.9/月 | 全部功能 + 监控 + 回滚 |
| **企业版** | 按需 | 私有部署 + 批量管理 |

### 4.2 变现路径

```
第一阶段（开源）
├── 开源核心功能，建立用户基础
├── 收集反馈，优化产品
└── 建立社区

第二阶段（SaaS）
├── 推出云服务版本
├── AI 分析收费
└── 增值服务（企业支持、定制开发）

第三阶段（生态）
├── 插件市场（第三方工具集成）
├── API 开放平台
└── 与硬件厂商合作预装
```

### 4.3 竞品分析

| 产品 | 优势 | 劣势 |
|------|------|------|
| CCleaner | 老牌，用户多 | 不支持迁移，AI 能力弱 |
| SpaceSniffer | 可视化好 | 功能单一，无 AI |
| Windows 存储感知 | 系统自带 | 功能有限，不智能 |
| **我们的产品** | AI 驱动 + 迁移能力 | 新产品，用户基础弱 |

### 5.2 关键设计

| 设计点 | 方案 |
|--------|------|
| AI 抽象层 | 支持 OpenAI/DeepSeek/Ollama 切换 |
